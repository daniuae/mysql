E-commerce Data and Common Table Expressions (CTEs)
This hands-on lab is designed to help you practice using Common Table Expressions (CTEs) in MySQL with a typical e-commerce dataset.
Part 1: Database Setup and Data Insertion
Execute the following SQL script to create the database, tables, and populate them with sample e-commerce data.
SQL
Create the database
CREATE DATABASE IF NOT EXISTS ecommerce_lab;
USE ecommerce_lab;

1. Customers Table
CREATE TABLE IF NOT EXISTS Customers (
   customer_id INT PRIMARY KEY,
   customer_name VARCHAR(100) NOT NULL,
   city VARCHAR(50)
);

2. Products Table
CREATE TABLE IF NOT EXISTS Products (
   product_id INT PRIMARY KEY,
   product_name VARCHAR(100) NOT NULL,
   category VARCHAR(50),
   price DECIMAL(10, 2) NOT NULL
);

3. Orders Table
CREATE TABLE IF NOT EXISTS Orders (
   order_id INT PRIMARY KEY,
   customer_id INT,
   order_date DATE NOT NULL,
   total_amount DECIMAL(10, 2) NOT NULL,
   FOREIGN KEY (customer_id) REFERENCES Customers(customer_id)
);

4. Order_Items Table (Details of what was in each order)
CREATE TABLE IF NOT EXISTS Order_Items (
   order_item_id INT PRIMARY KEY,
   order_id INT,
   product_id INT,
   quantity INT NOT NULL,
   unit_price DECIMAL(10, 2) NOT NULL,
   FOREIGN KEY (order_id) REFERENCES Orders(order_id),
   FOREIGN KEY (product_id) REFERENCES Products(product_id)
);

Part 2: CTE Practice.
Use Common Table Expressions (CTEs)defined with the WITH clause to solve the following business problems.
Challenge 1: Calculating Customer-Level Total Spend (Simple CTE)
Goal: Find the total revenue generated by each customer.
Create a CTE named CustomerTotalOrders to aggregate total spending per customer.
Join the CTE with the Customers table.
Solution 1:
SQL
WITH CustomerTotalOrders AS (
   1. CTE: Calculate total amount spent per customer
   SELECT
       customer_id,
       SUM(total_amount) AS total_spent
   FROM
       Orders
   GROUP BY
       customer_id
)
2. Main Query: Join CTE with Customers table
SELECT
   c.customer_name,
   c.city,
   t.total_spent
FROM
   CustomerTotalOrders t
JOIN
   Customers c ON t.customer_id = c.customer_id
ORDER BY
   t.total_spent DESC;
Challenge 2: Average Order Value by City (Multiple CTEs)
Goal: Determine the Average Order Value (AOV) for customers in each city.
Create CityOrders to link orders to cities.
Create CityAOV to calculate the average using the first CTE.
Solution 2:
SQL
WITH CityOrders AS (
   1. First CTE: Get order details with the customer's city
   SELECT
       o.order_id,
       o.total_amount,
       c.city
   FROM
       Orders o
   JOIN
       Customers c ON o.customer_id = c.customer_id
),
CityAOV AS (
   2. Second CTE: Calculate Average Order Value (AOV) per city from the first CTE
   SELECT
       city,
       AVG(total_amount) AS average_order_value
   FROM
       CityOrders
   GROUP BY
       city
)
3. Main Query: Select results from the second CTE
SELECT
   city,
   ROUND(average_order_value, 2) AS AOV
FROM
   CityAOV
ORDER BY
   average_order_value DESC;


Part 3: Recursive CTE.
Challenge 3: Tracing Product Category Hierarchy
Goal: Create a hierarchical structure of product categories and use a Recursive CTE (RCTE) to list the full path from the top-level parent to the lowest subcategory.
First, let's create and populate a new table for the hierarchy.
SQL
Create Product Hierarchy Table
CREATE TABLE IF NOT EXISTS Product_Hierarchy (
   category_id INT PRIMARY KEY,
   category_name VARCHAR(50) NOT NULL,
   parent_category_id INT,
   FOREIGN KEY (parent_category_id) REFERENCES Product_Hierarchy(category_id)
);
Insert Sample Hierarchy Data

(1, 'Electronics', NULL),      Top Level
(2, 'Computers', 1),           Child of Electronics
(3, 'Peripherals', 1),         Child of Electronics
(4, 'Laptops', 2),             Child of Computers

(6, 'Mice', 3),                Child of Peripherals

The Task: Use a Recursive CTE named CategoryPathFinder to trace the path from the top-level category down to every subcategory, outputting the full hierarchical path.
Solution 4 (Recursive CTE):
SQL
WITH RECURSIVE CategoryPathFinder AS (
   1. ANCHOR MEMBER (Starting Point): Select the top-level categories (where parent_category_id is NULL)
   SELECT
       category_id,
       category_name,
       parent_category_id,
       CAST(category_name AS CHAR(200)) AS category_path, -- Initialize the path column
       1 AS level
   FROM
       Product_Hierarchy
   WHERE
       parent_category_id IS NULL
   UNION ALL
   2. RECURSIVE MEMBER: Join the CTE with the table to find the children
   SELECT
       ph.category_id,
       ph.category_name,
       ph.parent_category_id,
       CONCAT(cpf.category_path, ' -> ', ph.category_name), -- Append the child category name to the path
       cpf.level + 1
   FROM
       Product_Hierarchy ph
   INNER JOIN
       CategoryPathFinder cpf ON ph.parent_category_id = cpf.category_id
)
3. Final SELECT Statement: Output the results
SELECT
   category_id,
   category_name,
   category_path,
   level
FROM
   CategoryPathFinder
ORDER BY
   category_path;
The query will output the full path for each category: | category_id | category_name | category_path | level | | :---: | :---: | :---: | :---: | | 1 | Electronics | Electronics | 1 | | 2 | Computers | Electronics -> Computers | 2 | | 5 | Desktops | Electronics -> Computers -> Desktops | 3 | | 4 | Laptops | Electronics -> Computers -> Laptops | 3 | | 3 | Peripherals | Electronics -> Peripherals | 2 | | 7 | Keyboards | Electronics -> Peripherals -> Keyboards | 3 | | 6 | Mice | Electronics -> Peripherals -> Mice | 3 |
